¶评价指标

T表示预测正确 F表示预测错误

P表示预测为正样本 N表示预测为负样本

预测为正 预测为负
实际为正 TP FN
实际为负 FP TN

问：给定数据集，模型预测的正样本总数有多少？
答：TP + FP
问：数据集里真正的正样本总数有多少？
答：TP + FN

¶TPR

真正例率（灵敏度/召回率）：指实际为正样本中被正确预测为正样本的比例。它表示模型对正样本的识别能力。公式：TPR = TP / (TP + FN)。

TPR衡量模型对正样本的敏感度，召回的正样本比例

¶FPR

假正例率：指实际为负样本中被错误预测为正样本的比例。它衡量了模型误判负样本为正样本的概率。公式：FPR = FP / (FP + TN)。

FPR 表示模型错误地将负样本判断为正样本的可能性，越低越好。

¶TNR

真负例率（特异性）：指实际为负样本中被正确预测为负样本的比例。它反映了模型正确排除负样本的能力。公式：TNR = TN / (TN + FP) = 1 - FPR。

TNR 反映模型正确识别负样本的能力，越高越好。

¶FNR

假负例率：指实际为正样本中被错误预测为负样本的比例。它表示漏掉的正样本比例。公式：FNR = FN / (FN + TP) = 1 - TPR。

FNR 则是模型未能识别出正样本的错误率，数值越低说明模型性能越好。

¶Precision

精确率 是分类问题中的一个重要指标，特别在二分类任务中。它表示模型预测为正的样本中，实际为正样本的比例，反映了模型在正类预测中的准确性。公式：Precision = TP / (FP + TP)。

¶F1- score

虽然Precision 和 Recall 的值我们预期是越高越好，但是这两个值在某些场景下却是存在互斥的，比如仅仅取一个样本，并且这个样本也确实是正样本，那么Precision = 1.0， 然而 Recall 可能就会比较低（在该样本集中可能存在多个样本）；相反，如果取所有样本，那么Recall = 1.0，而Precision就会很低了。所以在这个意义上，该两处值需要有一定的约束变量来控制。

F1-Score特别适用于当我们希望在正负样本不平衡的情况下，平衡模型的精确率和召回率。

它取值范围在0到1之间，1表示模型的精确率和召回率都达到了最佳效果。

¶ROC

ROC曲线图有两个维度，横轴表示FPR，纵轴表示TPR，用于描述二元分类在不同阈值下的表现。ROC表示了在收益（true positives）和成本（false positives）之间的权衡关系。ROC曲线越靠近左上角，表示模型性能越好，因为此时TPR较高而FPR较低。

ROC 可以反映二分类器的总体分类性能，但是无法直接从图中识别出分类最好的阈值，事实上最好的阈值也是视具体的场景所定；

ROC曲线一定是需要在 y = x之上的，否则就是一个不理想的分类器；

¶AUC

AUC是ROC曲线下的面积，取值范国从0到1，其中0.5表示模型预测效果等同于随机猜测，1表示完美分类器。AUC越接近1，表示模型具有更好的性能。

AUC的优点在于它不受分类阈值的影响，可以综合考虑模型在不同阈值下的性能表现。此外，AUC对于样本不平衡的情况下也比较鲁棒，不会受到正负样本比例的影响。

¶物理意义

分类模型对正样本和负样本的排列顺序判别能力。

假设有一个二元分类模型，它将样本划分为正类和负类，并输出相应的概率值或分数。AUC表示的是当我们随机选择一个正样本和一个负样本时，模型正确地将正样本的概率值或分数排在负样本之前的概率。

换句话说，**AUC反映了模型正确预测正样本排在负样本前面的能力。**如果AUC接近于1，那么模型很好地区分了正样本和负样本，几乎所有正样本的概率值或分数都高于负样本。而如果AUC接近于0.5，说明模型的分类能力等同于 随机猜测，无法有效区分正负样本。

在实际应用中，AUC可以被解释为模型的“区分度”或“准确性”。一个具有较高AUC值的模型更有可能在真实应用中能够正确地将正样本与负样本区分开来。

需要注意的是，AUC本身并不提供具体的预测概率或分数阈值，而是综合了所有可能的國值下的分类性能。因此，AUC在比较不同模型的分类性能时非常有用，尤其是当具体的阈值选择对问题的背景不敏感时。

¶如何使用非参数法手动计算AUC

使用非参数法计算AUC时，不依赖具体的概率分布假设或模型假设，而是通过对样本的排序估算AUC

  1. 获取模型的预测值

首先，通过分类模型（例如逻辑回归、支持向量机或随机森林等），获取每个样本的概率分数。

  1. 排序样本

将样本按照预测的概率值从高到低进行排序。假设有 N 个样本，我们将这些样本根据预测值或分数从高到低进行排列。这样可以确保预测为正的样本在前，预测为负的样本在后。

  1. 统计正样本在负样本前面的次数

遍历排序后的样本列表，统计每一个正样本排在负样本前面的次数。这可以通过以下方式完成：当遇到一个正样本时，记录它前面已经遇到的负样本数。如果遇到负样本，则不做任何操作。这个过程可以累积出正样本相对于负样本的优势排序情况。

  1. 计算正样本排在负样本前的比例

将上一步记录的负样本数量除以总负样本数，即可得出正样本排在负样本前的比例。此比例用于衡量模型分类的有效性。

  1. 计算AUC值

正样本排在负样本前的比例即为AUC值。

我们可以看到，正样本的排序相对于负样本如下：

  • 样本1（正）排在2个负样本前（样本2，样本4）。
  • 样本3（正）排在1个负样本前（样本4）。
  • 样本5（正）没有排在任何负样本前。

¶线上和线下AUC不一致可能的原因
  1. 特征穿越：指模型在训练时使用了与标签强相关的特征。这种情况下，模型在训练集上的表现可能非常好，但在实际应用中，由于这种特征不具备普遍性，模型效果会大幅下降。
  2. 数据分布不一致：模型在训练和测试时所使用的数据特征不同。例如，离线训练时使用的某个特征长度为50，但在上线时该特征长度变为30。这种特征维度的差异会导致模型线上表现不如线下。
  3. 冰山效应：在训练和测试过程中，模型往往会针对“点击正样本”和“未点击负样本”进行优化。在离线测试时，效果可能表现得很好，但实际上模型只是在某些特定的曝光样本上表现优异，即类似于冰山的“顶部”。然而，线上应用时需要面对更广泛的用户群体和全样本数据，这种情况下表现可能会不如预期。
¶用途

ROC曲线和AUC在实际应用中有多种用途，包括但不限于以下几个方面：

  • 模型评估与比较：ROC曲线和AUC可以用来评估和比较不同分类模型的性能。通过比较不同模型的ROC曲线和AUC值，可以确定哪个模型在给定任务上表现更好。

  • 阈值选择：在某些情况下，需要根据特定的需求调整分类器的阈值。ROC曲线可以帮助理解在不同阈值下的真正例率和假正例率之间的权衡关系，从而选择最佳的阈值。

  • 特征选择：ROC曲线和AUC也可以用于评估特征的重要性。通过观察在不同特征组合下的模型性能，可以识别对分类结果影响最大的特征。

  • 不平衡数据集的评估：在处理不平衡数据集（例如正负样本比例差异很大）时，准确度等常用评估指标可能不足以反映模型性能。ROC曲线和AUC对于不平衡数据集更具有鲁棒性，能够更好地评估模型的分类能力。

  • 模型调优：在机器学习模型的训练过程中，可以使用ROC曲线和AUC来指导模型的调优策略，例如调整模型的超参数或改进特征工程。
